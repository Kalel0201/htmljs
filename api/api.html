Data

1. Get and set

html.data give you ability to observe any kind of object - from basic types like String Number Boolean Array and Function.

You want to observe a string. 

var helloWorld = html.data('Hello world!');
Get the value of helloWorld variable. 
//get value
helloWorld(); // "Hello world!"
Set the value of helloWorld variable. 
//set value
hellowWorld('Hello HTML Engine');
Other type to observe. 
var a = html.data(1),
    b = html.data(2),
    computed = html.data(function() {
        return a() + b();
    });
    //computed() => 3

var array = html.data([1,2,3,4,5]);

2. changeAfter

-method

Register dependency of a computed value.

ParamTypeDetails
arguments html.data

Register as many dependencies as you want.

 
var firstName = html.data('Henry'),
    lastName = html.data('Ford'),
    fullName = html.data(function() {
        return firstName() + ' ' + lastName();
    });

//fullName() ==> "Henry Ford"

3. delay

-property

Set delay time to notify change to UI and its dependencies. 

var test = html.data('Someone');
    test.delay = 100;
//now every time an input change test, all UI and its dependencies will be notified after 100 milliseconds.

4. dependencies

-property

Get all dependencies from html.data. 

var a = html.data(4),
b = html.data(2);
c = html.data(function() {
    return a*b;
}).changeAfter(a, b);

//c.dependencies ==> [a,b]

5. isComputed

Check the data is kind of function (aka computed).

var firstName = html.data('Henry'),
    lastName = html.data('Ford'),
    fullName = html.data(function() {
        return firstName() + ' ' + lastName();
    });
//lastName.isComputed() ==> false
//fullName.isComputed() ==> true

6. lazyInput

-property

Set this property for indicate when the observed value change.
If true then the value only change after change event.
If false then the value will change immediately when key stroke.

7. Refresh

-method

Use refresh method for update data manually, especially when working with complex computed values.

hellowWorld.refresh();

8. setValidationResult

-method

Use to set validation result.

ParamTypeDetails
isValid Boolean

True if the data is valid.
message String

Optional. Message to show when data is not valid.

 
// custom validation rule
html.data.validation.beginWith = function(char, message) {
    this.validate(function(newValue, oldvalue) {
        if(newValue.charAt(0).toLowerCase() !== char.toLowerCase()) {
            this.setValidationResult(false, message);
        } else {
            this.setValidationResult(true);
        }
    });
    return this;
};
// custom validation rule usage
var name = html.data('Someone').beginWith('a');

9. silentSet

-method

Use to set value without notifying UI and dependencies.

ParamTypeDetails
value Object

The new value.

 
var name = html.data('Someone');
name.silent('Another name');

10. subscribe

-method

Subscribe a function to be run when the value changed.

ParamTypeDetails
arg Function<object, object>

Function to run when the value of observer changed.

 
//set value
hellowWorld.subscribe(function(value, lastValue) {
    console.log(value, lastValue);
});

11. targets

-property

All functions that subscribed to this observer.

12. unsubscribe

-method

Unsubscribe a function from observer. This method will remove a function in html-data-targets

var firstName = html.data('Henry'),
sub = function(val) {
    console.log(val);
};
firstName.subscribe(sub);
// firstName.targets.length ==> 1
firstName.unsubscribe(sub);
// firstName.targets.length ==> 0

13. validate

-property

Use to create custom validation rules.

ParamTypeDetails
arguments Function<newValue, oldValue>

The callback function will have 2 parameters: oldValue and newValue.
See the example in setValidationResult

14. validators

-property

All validators of the observer.

15. validationResults

-property

All validationResults of the observer.

Data <Array>

You can only use these methods as following when you need to observe an array.
All the methods in html.data above are available for html.data<Array>

1. add

-method

Add an object to the list, also update the UI/dependencies.

ParamTypeDetails
object Object

New object to add.
index Number

(Optional - default 0) Index for new item.

 
var list = html.data([1,2,3,4,5,6]);
list.add(7);
list.add(0, 0);

2. filter

-method -no return

Filter a list and display the searching result.

ParamTypeDetails
searchString String

Search string.

 
var list = html.data(['Anderson', 'Peter', 'Robin']);
list.filter('an'); // ==> ["Anderson"]
list.filter('er'); // ==> ["Anderson", "Peter"]

3. getFilterResult

-method -return Array filtered list

Return the filter result.

var list = html.data(['Anderson', 'Peter', 'Robin']);
list.filter('an'); list.getFilterResult(); // ==> ["Anderson"]
list.filter('er'); list.getFilterResult(); // ==> ["Anderson", "Peter"]

4. move

-method -no return

Move an item in the list to a new position, also update the UI/dependencies.

ParamTypeDetails
position Number

Position of an item.
newPosition Number

New position to move to.

 
var list = html.data(['Anderson', 'Peter', 'Robin']);
list.move(0, 2);
//list() ==> ['Peter', 'Robin', 'Anderson']

5. orderBy

-method -return html.data

Order a list by some criteria, also update the UI/dependencies. Support multiple level order.

ParamTypeDetails
arguments String Function Array

You can pass one of these types to order the list.

 
var people = html.data([
    {name: 'EEE', age: 20},
    {name: 'ABC', age: 50},
    {name: 'BBB', age: 25}
]);
people.orderBy('name'); // ==> [{name: 'ABC', age: 50}, {name: 'BBB', age: 25}, {name: 'EEE', age: 20}]
people.orderBy('age'); // ==> [{name: 'EEE', age: 20}, name: 'BBB', age: 25}, {name: 'ABC', age: 50}]
people.orderBy({field: 'name', isAsc: false}); // ==> [{name: 'EEE', age: 20}, name: 'BBB', age: 25}, {name: 'ABC', age: 50}]
people.orderBy({field: 'age', isAsc: false}); // ==> [{name: 'ABC', age: 50}, {name: 'BBB', age: 25}, {name: 'EEE', age: 20}]
people.orderBy([{field: 'age', isAsc: false}, {field: 'age', isAsc: true}]); // ==> [{name: 'ABC', age: 50}, {name: 'BBB', age: 25}, {name: 'EEE', age: 20}]

var orderFn = function(x) {return x.name; };
orderFn.isAsc = false;
people.orderBy(orderFn); // equal to people.orderBy({field: 'name', isAsc: false});

6. pop

-method -return html.data

Remove the last item in the observed list, also update the UI/dependencies.

var people = html.data([{name: 'Brad Pitt', age: 46}, {name: 'Angelina Jolie', age: 34}]);
people.pop();
// people() ==> [{name: 'Brad Pitt', age: 46}]

7. push

-method -return html.data

Add a new item at the last position in the list, also update the UI/dependencies..

ParamTypeDetails
obj Object

Should be the same type within the list.

 
var people = html.data([{name: 'Brad Pitt', age: 46}]);
people.push({name: 'Angelina Jolie', age: 34});
// people() ==> [{name: 'Brad Pitt', age: 46}, {name: 'Angelina Jolie', age: 34}]

8. remove

-method -return html.data

Remove an item in the list, also update the UI/dependencies.

ParamTypeDetails
obj Object

The item in the list.

 
var people = html.data([{name: 'Brad Pitt', age: 46}, {name: 'Angelina Jolie', age: 34}]);
people.remove(people()[0]);
// people() ==> [{name: 'Angelina Jolie', age: 34}]

9. removeAt

-method -return html.data

Remove an item by its index in the list, also update the UI/dependencies.

ParamTypeDetails
index Number

The index of item in the list.

 
var people = html.data([{name: 'Brad Pitt', age: 46}, {name: 'Angelina Jolie', age: 34}]);
people.removeAt(0);
// people() ==> [{name: 'Angelina Jolie', age: 34}]

10. setFilterResult

-method -no return.

Set the filter result by yourself (using another algorithm instead of default algorithm), keep the original array inside, just update the UI using the result.

ParamTypeDetails
filterResult Array<Object>

Required. An array to display. Items in array must be the same type/structure to item in the observed list.
You can clear the filter result manually and display the original array by passing null param. 

var people = html.data([{name: 'Brad Pitt', age: 46}, {name: 'Jennifer Aniston', age: 43}]);
people.setFilterResult([{name: 'Angelina Jolie', age: 34}]);
// people.getFilterResult() ==> [{name: 'Angelina Jolie', age: 34}]

11. splice

-method -no return.

Remove items/add new items, also update the UI/dependencies.

ParamTypeDetails
index Number

Required. An integer that specifies at what position to add/remove items.
howmany Number

Required. The number of items to be removed. If set to 0, no items will be removed.
items Object Array<Object>

Optional. New items to be added, it can be an item or a list.
If you pass an array, the observed list will add each item one by one. It's slightly different from splice of Array prototype. 

var people = html.data([{name: 'Brad Pitt', age: 46}, {name: 'Jennifer Aniston', age: 43}]);
people.splice(1, 1, {name: 'Angelina Jolie', age: 34});
// people() ==> [{name: 'Brad Pitt', age: 46}, {name: 'Angelina Jolie', age: 34}]

people.splice(1, 0, [{name: 'Pax Thien', age: 12}, {name: 'Zahara Marley', age: 12}]);
// people() ==> [{name: 'Brad Pitt', age: 46}, {name: 'Angelina Jolie', age: 34}, {name: 'Pax Thien', age: 12}, {name: 'Zahara Marley', age: 12}]

12. swap

-method -no return.

Swap two items in the list, using their indexes, also update the UI/dependencies..

ParamTypeDetails
firstIndex Number

The first index of the first item need to swap.

 
var people = html.data([{name: 'Brad Pitt', age: 46}, {name: 'Angelina Jolie', age: 34}]);
people.swap(0, 1);
// people() ==> [{name: 'Angelina Jolie', age: 34}, {name: 'Brad Pitt', age: 46}]

13. where

-method -return html.data

Filter the observed list using expression, also update UI/dependencies using the result.

ParamTypeDetails
callback Function<Object>

Required. The callback using to filter, the callback takes each object in the observed list as parameter.

 
var people = html.data([{name: 'Brad Pitt', age: 46}, {name: 'Jennifer Aniston', age: 43}]);
people.where(function(person) {return person.name == 'Jennifer Aniston';});
// people.getFilterResult() ==> [{name: 'Jennifer Aniston', age: 43}]

Validation

You can use these built-in validation rules with fluent API.

1. equal

Compare to an another object, note that you can't compare to a reference type like object, array

ParamTypeDetails
compareTo String Number Date...

Required. The object to compare.
message String

Required. The message to show when user types invalid data.

 
var a = html.data(123).equal(123, 'Not equal to 123');

2. isNumber

Check the data is kind of number or string representing a number. Accept float and integer.

ParamTypeDetails
message String

Required. The message to show when user types something that's not a number/string of number.

 
var a = html.data(123).isNumber('You must enter a number.');

3. greaterThan

Check the data is greater than a criteria.

ParamTypeDetails
valueToCompare String Number Date

Required. The value to compare to the original data.
message String

Required. The message to show when data is not valid.

 
var a = html.data(123)
   .isNumber('You must enter a number.')
   .greaterThan(100, 'You must enter a number greater than 100.');

4. greaterThanOrEqual

Check the data is greater than or equal a criteria. Similar to greaterThan.

ParamTypeDetails
valueToCompare String Number Date

Required. The value to compare to the original data.
message String

Required. The message to show when data is not valid.

 
var a = html.data(123)
   .isNumber('You must enter a number.')
   .greaterThanOrEqual(100, 'You must enter a number greater than or equal 100.');

5. lessThan

Check the data is less than a criteria.

ParamTypeDetails
valueToCompare String Number Date

Required. The value to compare to the original data.
message String

Required. The message to show when data is not valid.

 
var a = html.data(123)
   .lessThan(100, 'You must enter a number less than 100.');

6. lessThanOrEqual

Check the data is less than or equal a criteria. Similar to lessThan.

ParamTypeDetails
valueToCompare String Number Date

Required. The value to compare to the original data.
message String

Required. The message to show when data is not valid.

 
var a = html.data(123)
   .lessThanOrEqual(100, 'You must enter a number less than 100.');

7. maxLength

Check the max length of a string.

ParamTypeDetails
length Number

Required. The maximum length allowed.
message String

Required. The message to show when data is not valid.

 
var password = html.data('123456')
   .maxLength(32, 'Max length of password is 32. Please shorten your password.');

8. minLength

Check the minimum length of a string.

ParamTypeDetails
length Number

Required. The maximum length allowed.
message String

Required. The message to show when data is not valid.

 
var username = html.data('my')
   .maxLength(32, 'Min length of username is 6.');

9. pattern

Check the string using regular expression.

ParamTypeDetails
reg Regular

Required. The regular expression used to validate data.
message String

Required. The message to show when data is not valid.

 
var username = html.data('my')
   .maxLength(32, 'Min length of username is 6.');

Extend html.data

1. Validation

You add more validation rules in html.data through html.data.validation namespace.
The function to validate data has as many parameters as you want, however you must call validate and setValidationResult methods in html.data once and only once.

See the example in setValidationResult
Another example. 
html.data.validation.genderAndAge = function(gender, age, message) {
    this.validate(function(newValue, oldValue) {  
        if (newValue.gender != gender || newValue.age < age) {
            this.setValidationResult(false, message);
        } else {
            this.setValidationResult(true);
        }
    });  
    return this;
};
Usage 
var person = html.data({name: 'Nhan Nguyen', gender: 'male', age: 25})
   .genderAndAge('male', 20, 'Require male and greater than 25 years old.');
